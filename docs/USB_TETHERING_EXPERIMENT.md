# v7 USB 網路共用實驗封存

v7.0.0 將 QuietPanel 從 ADB Port Forward 改為 Android USB 網路共用（RNDIS）直接 TCP 連線，目標是在日常使用時不需要開發者模式或 USB 偵錯。

## 結果

在 Xiaomi 2013023（紅米一代、Android 4.2.2）上，Bridge 與 App 可成功透過 `192.168.42.0/24` USB 網段連線；系統資料、巨集指令與 NASA 圖片通道均曾正常運作。

但長時間測試中，Windows 10 的事件紀錄多次出現 `Remote NDIS based Internet Sharing Device` 網路介面被移除／重設。當 RNDIS 介面消失時，Bridge 仍在執行，但沒有通道可傳送資料或接收巨集指令；介面重新建立後才會恢復。這個問題發生在 USB 網路共用／裝置驅動層，不是 QuietPanel 協定或畫面程式本身。

這台舊手機原本的 ADB 方案在相同長時間使用情境下穩定，且能配合 PC 螢幕同步省電功能。因此 v7 USB 網路共用不作為紅米一代的正式運行方案，主線回復 ADB 版本。

## 未來可再評估的方向

- 較新的平板使用同一 Wi-Fi 區域網路連線，而不是 USB 網路共用。
- 以較新的 Android 裝置重新驗證 USB 網路共用的長時間穩定性。
- 若使用 USB 網路共用，加入網路介面中斷的明確提示與復原流程；但無法在手機已切斷 USB 網路時由 Bridge 主動喚醒手機。

此分支保留 v7.0.0 實作與測試紀錄，供未來參考；不建議用於此紅米一代的常態展示。
